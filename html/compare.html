Installation
  • apt-get install make
  • apt-get install g++
  • apt-get install libssl-dev
  • apt-get install git
  • git clone https://github.com/gigablast/open-source-search-engine.git ./gb
  • cd gb
  • make
  • edit hosts.conf to set working-dir
  • ./gb 0
  • wait for binary data files to build
  • ./gb 0
  • point browser to http://127.0.0.1:8000/
  • config (gb.conf hosts.conf) and index stored in current working dir
  • sudo apt-get install solr-tomcat (188MB of additional space)
  • sudo service tomcat6 start
  • point browser to http://127.0.0.1:8080/solr
  • config and index stored at /usr/share/solr
    • wget https://download.elasticsearch.org/elasticsearch/elasticsearch/elasticsearch-0.90.3.zip
    • unzip elasticsearch-0.90.3.zip
    • cd elasticsearch-0.90.3
    • cd bin
    • ./elasticsearch -f
    • curl -X GET http://localhost:9200/
    Injecting documents via cmdline ./gb inject <filename> 127.0.0.1:8000
    Each page in the file must be delineated by +++URL:http://myurl.com/xxx\n. Use the supplied file injectme3 which has 454 HTML docs in it for testing. i.e. Start up a gigablast instance and type ./gb inject injectme3 127.0.0.1:8000 to see it in action. It currently only reads up to 1MB at a time so you will have to change doInject() in main.cpp if you have larger html documents than that in the file.
